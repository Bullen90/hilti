Instruction Reference¶

In the following, we list the complete set of instructions that the HILTI language provides. For reference, we give them with both their textual mnemonics (as one uses them writing a HILTI program in a text editor), along with their Python class names as provided by the HILTI Python module.

Control Flow Operations¶

class hilti.instructions.flow.Call(op1, op2, target)¶
<any> = call <function> <(tuple)>

Calls function using the tuple in op2 as arguments. The argument types as well as the target’s type must match the function’s signature.

class hilti.instructions.flow.CallC(op1, op2, target)¶
<any> = call.c <function> <(tuple)>

For internal use only.

Calls the external C function using the tuple in op2 as arguments. The argument types as well as the target’s type must match the function’s signature.

class hilti.instructions.flow.CallTailResult(op1, op2, op3, target)¶
<any> = call.tail.result <function> <(tuple)> <label>

Like Call(), calls function using the tuple in op2 as arguments. The argument types must match the function’s signature. The function must return a result value and it’s type must likewise match the function’s signature. Different than Call(), CallTailVoid must be the last instruction of a Block. After the called function returns, control is passed to block op3.

This instruction is for internal use only. During the code generation, all Call instructions are converted into either CallTailVoid or CallTailResult instructions.

class hilti.instructions.flow.CallTailVoid(op1, op2, op3)¶
call.tail.void <function> <(tuple)> <label>

For internal use only.

Calls function using the tuple in op2 as arguments. The argument types must match the function’s signature, and the function must not return a value. Different than Call(), CallTailVoid must be the last instruction of a Block. After the called function returns, control is passed to block op3.

This instruction is for internal use only. During the code generation, all Call instructions are converted into either CallTailVoid or CallTailResult instructions.

class hilti.instructions.flow.IfElse(op1, op2)¶
if.else <bool> <(label, label)>

Jumps to label op2[0] if op1 is true, otherwise to op2[1].

class hilti.instructions.flow.Jump(op1)¶
jump <label>

Jumps unconditionally to label op2.

class hilti.instructions.flow.ReturnResult(op1)¶
return.result <int:* or None>

Returns from the current function with the given value. The type of the value must match the function’s signature.

class hilti.instructions.flow.ReturnVoid¶
return.void

Returns from the current function without returning any value. The function’s signature must not specifiy a return value.

Integer Operations¶

class hilti.instructions.integer.Add(op1, op2, target)¶
<int> = integer.add <int> <int>

Calculates sum of two operands. Operands and target must be of same width.

class hilti.instructions.integer.Div(op1, op2, target)¶
<int> = integer.div <int> <int>

Divides op1 by op2, flooring the result. Operands and target must be of same width.

Throws Exception.DivisionByZero if op2 is zero.

class hilti.instructions.integer.Mult(op1, op2, target)¶
<int> = integer.mult <int> <int>

Multiplies op1 with op2. Operands and target must be of same width.

class hilti.instructions.integer.Sub(op1, op2, target)¶
<int> = integer.sub <int> <int>

Subtracts op1 one from op2. Operands and target must be of same width.
